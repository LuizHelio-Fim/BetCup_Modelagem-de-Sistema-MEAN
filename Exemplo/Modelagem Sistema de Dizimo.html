Os dados são armazenados em MongoDB (collection offerings). O admin visualiza via rota autenticada com role.
Fluxo de dados (resumo técnico)
  1. Membro se cadastra → login → vê QRCode com chave suaigreja@igreja.net.
  2. Membro realiza o PIX (externo ao sistema) e depois preenche o formulário de oferta informando valor + motivação.
  3. Backend (Node + MongoDB) registra oferta vinculada ao membro (relação memberId).
  4. Pastor (admin) visualiza total de todas as ofertas e a lista completa, podendo acompanhar motivações e valores.

Arquitetura: Frontend Angular SPA, API REST com Express, driver nativo MongoDB (sem Mongoose). Autenticação JWT com guardas de rota. Layout totalmente responsivo e empilhado.
